## 3. 数据标准体系

### 3.1 数据标准分层

| 标准类型 | 内容 | 示例 | 强制级别 |
|----------|------|------|----------|
| 基础标准 | 命名规范、编码规则 | `表名: ODS_主题_源系统_表名` | 强制 |
| 数据元标准 | 字段定义、值域范围 | `客户年龄: 0-150, 整数` | 强制 |
| 代码标准 | 统一代码映射 | `性别: M/F/U` | 强制 |
| 交换标准 | 接口格式、传输协议 | `JSON + HTTPS` | 强制 |
| 质量标准 | 准确性/完整性/及时性指标 | `准确率 ≥ 99.5%` | 建议 |

### 3.2 命名规范示例

```sql
-- 表命名规范：{层级}_{主题}_{描述}_{分区后缀}
-- 层级: ODS(贴源) / DWD(明细) / DWS(汇总) / ADS(应用)

-- ✅ 规范命名
CREATE TABLE dwd_trade_order_detail_di (
    id              BIGINT        COMMENT '主键ID',
    order_id        VARCHAR(32)   COMMENT '订单编号',
    user_id         BIGINT        COMMENT '用户ID',
    product_id      BIGINT        COMMENT '商品ID',
    pay_amount      DECIMAL(16,2) COMMENT '实付金额（元）',
    order_status    VARCHAR(16)   COMMENT '订单状态',
    create_time     DATETIME      COMMENT '创建时间',
    dt              VARCHAR(10)   COMMENT '分区日期'
) COMMENT '交易域-订单明细表-日增量'
PARTITIONED BY (dt);
```

## 4. 数据分级分类

### 4.1 分类矩阵

| 安全等级 | 定义 | 数据示例 | 加密要求 | 访问控制 | 审计要求 |
|----------|------|----------|----------|----------|----------|
| **L4 绝密** | 泄露造成特别严重损害 | 密钥、根证书 | 必须加密 | 最小范围授权 | 实时审计 |
| **L3 机密** | 泄露造成严重损害 | 客户 PII、财务报表 | 必须加密 | RBAC + 审批 | 实时审计 |
| **L2 内部** | 泄露造成一般损害 | 员工通讯录、项目文档 | 建议加密 | RBAC | 定期审计 |
| **L1 公开** | 泄露无损害 | 产品说明、公开报告 | 不要求 | 公开 | 不要求 |

### 4.2 个人敏感信息处理规则

```yaml
处理规则:
  PII (个人身份信息):
    姓名: 加密存储，显示时脱敏（李**）
    身份证号: 强加密，禁止明文显示
    手机号: 加密存储，显示时脱敏（138****8000）
    邮箱: 加密存储，保留@后部分
    地址: 脱敏至区县级（上海市浦东新区**）
  
  生物特征:
    人脸: 强加密，独立存储，禁止导出
    指纹: 单向哈希，不可逆
  
  金融信息:
    银行卡号: 强加密，仅保留后4位
    交易密码: 单向哈希 + 盐
```

## 5. 数据血缘与元数据

### 5.1 血缘追踪场景

```
数据流转追踪示例（客户 360 视图场景）：

CRM 客户表 ──┐
POS 交易表 ──┼──→ DWD 客户交易宽表 ──→ ADS 客户画像 ──→ BI 看板
APP 行为表 ──┘              │
                            └──→ ADS RFM 分析 ──→ 营销系统

影响分析：
  当 CRM 客户表新增字段 "会员等级"：
    1. DWD 客户交易宽表 → 需要同步更新
    2. ADS 客户画像 → 可以纳入新字段
    3. ADS RFM 分析 → 不受影响
```

## 6. 数据质量管理

### 6.1 质量维度

| 维度 | 定义 | 度量方式 | 目标值 |
|------|------|----------|--------|
| 完整性 | 数据是否缺失 | 非空率 = 非空行数/总行数 | ≥ 99.9% |
| 准确性 | 数据是否真实可靠 | 校验通过率 | ≥ 99.5% |
| 一致性 | 跨系统数据是否一致 | 交叉验证通过率 | ≥ 99% |
| 及时性 | 数据更新是否及时 | 数据延迟时间 | < 1h (T+1) |
| 唯一性 | 数据是否唯一不重复 | 重复率 = 重复行/总行数 | < 0.1% |

## 7. 合规管理

### 7.1 核心法规对照

| 法规 | 核心要求 | 企业应对措施 |
|------|----------|--------------|
| 《数据安全法》 | 数据分类分级、安全审查 | 建立数据分类分级制度，定期安全评估 |
| 《个人信息保护法》 | 知情同意、最小必要 | 完善隐私政策，实现数据最小化采集 |
| GDPR | 数据主体权利、DPIA | 跨境数据传输评估，DSAR 响应流程 |
| 等保 2.0 | 等级保护定级备案 | 三级系统等保测评，安全整改 |

### 7.2 数据跨境传输检查清单

- [ ] 是否涉及个人信息出境？
- [ ] 是否完成个人信息保护影响评估（PIA）？
- [ ] 是否与境外接收方签署标准合同？
- [ ] 是否通过网信办安全评估（如触发条件）？
- [ ] 是否建立了数据出境安全管理制度？
